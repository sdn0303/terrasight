03 総合ロケーションスコア (TLS)

地価公示ポイント（L01）を分析の起点とし、各ポイントに対して空間結合で周辺データを集約する。スコアはすべて 0-100 に正規化し、加重合計で TLS を算出する。全軸「高い = 良い」で統一。

TLS = 0.25 × S1災害 + 0.15 × S2地盤 + 0.25 × S3利便性 + 0.15 × S4将来性 + 0.20 × S5価格

S1 災害リスク

Weight 25%
洪水・液状化・地震・津波・土砂災害の複合評価。min()ペナルティで致命リスクを反映。

S2 地盤・地形

Weight 15%
AVS30 + 地形分類 + 地質。Phase 1 では AVS30 単独運用。

S3 生活利便性

Weight 25%
トランジット（駅距離×乗降客数）+ 教育 + 医療のアクセシビリティ。

S4 将来性

Weight 15%
人口増減率 × 乗降客数トレンド × 容積率余裕。

S5 価格分析

Weight 20%
用途地域内偏差値（割安判定）+ 取引活性度。

津波 (F_tsun) / 土砂災害 (F_land)

津波: 浸水深 0m=100, <0.3m=85, 0.3-1m=60, 1-2m=35, >2m=10。土砂: 該当なし=100, 警戒区域=40, 特別警戒=10。

軸1合成式

S1 = min(F_flood, F_liq, F_seis, F_tsun, F_land) × (0.3×F_flood + 0.25×F_liq + 0.25×F_seis + 0.1×F_tsun + 0.1×F_land) / 100
設計意図

加重平均だけだと「洪水危険だが地震は安全」で中間点に収まる。min() を掛けることで 1つでも深刻なリスクがあれば全体スコアが大幅に下がる 構造にしている。

06 軸3: 生活利便性スコア (S3) — Weight 25%

トランジットスコア (L_transit)

nearest_dist = 最寄り駅までの距離(m)
distance_factor = max(0, 100 − nearest_dist / 8)
ridership_factor = min(100, log10(best_ridership) × 20)
multi_line_bonus = (1km以内の路線数 − 1) × 10

L_transit = min(100, distance_factor×0.5 + ridership_factor×0.3 + multi_line_bonus×0.2)

教育スコア (L_edu)

L_edu = min(100, school_count_800m × 12 + diversity_bonus)
diversity_bonus = (小学校あり + 中学校あり) × 15

医療スコア (L_med)

L_med = min(100, hospital_count×20 + clinic_count×5 + log10(total_beds+1)×10)
S3 = 0.45×L_transit + 0.25×L_edu + 0.30×L_med
Phase 1（駅未実装時）: S3 = 0.45×L_edu + 0.55×L_med

07 軸4: 将来性スコア (S4) — Weight 15%

人口動態 (P_pop)

500mメッシュ 2020→2050 増減率: +20%以上=100, +10-20%=85, 0-+10%=65, -10-0%=40, -20--10%=20, <-20%=5

地価トレンド (P_price)

cagr = (price_2024 / price_2020)^(1/4) − 1
P_price = clamp(50 + cagr × 500, 0, 100)

容積率余裕 (P_far)

P_far = min(100, designated_far / 8)
S4 = 0.40×P_pop + 0.35×P_price + 0.25×P_far

08 軸5: 価格分析スコア (S5) — Weight 20%

相対バリュースコア (V_rel)

同一用途地域内での㎡単価の偏差値（z-score）。z_score < 0 = 中央値以下 = 割安 → スコア高い。

V_rel = clamp(50 − z_score × 20, 0, 100)

取引活性度 (V_vol)

V_vol = min(100, tx_count_500m_1yr × 5)
S5 = 0.65×V_rel + 0.35×V_vol

09 クロス分析パターン

バリューディスカバリー

value = S1 × (100 − V_rel) / 100
安全なのに割安 = 市場の見落とし

需要予測

demand = S3 × S4 / 100
利便性高 × 人口増 = 強い需要

地面の総合安全性

ground = S1 × S2 / 100
災害リスク × 地盤品質の複合評価

12 データ投入テンプレート

液状化テーブル

CREATE TABLE IF NOT EXISTS liquefaction (
  id SERIAL PRIMARY KEY,
  pl_class TEXT,
  pl_value NUMERIC,
  geom GEOMETRY(MultiPolygon, 4326)
);
CREATE INDEX idx_liquefaction_geom ON liquefaction USING GIST(geom);

駅テーブル

CREATE TABLE IF NOT EXISTS stations (
  id SERIAL PRIMARY KEY,
  station_name TEXT,
  operator TEXT,
  line_name TEXT,
  ridership INTEGER,
  year INTEGER,
  geom GEOMETRY(Point, 4326)
);
CREATE INDEX idx_stations_geom ON stations USING GIST(geom);

人口メッシュテーブル

CREATE TABLE IF NOT EXISTS population_mesh (
  id SERIAL PRIMARY KEY,
  mesh_code TEXT,
  pop_2020 INTEGER,
  pop_2050 INTEGER,
  change_rate NUMERIC,
  geom GEOMETRY(Polygon, 4326)
);
CREATE INDEX idx_pop_mesh_geom ON population_mesh USING GIST(geom);

Phase 1 MVP スコア算出クエリ

WITH point AS (
  SELECT geom FROM land_prices WHERE id = $1
),
flood_score AS (
  SELECT COALESCE(
    100 - MAX(CASE depth_rank
      WHEN 1 THEN 20 WHEN 2 THEN 50 WHEN 3 THEN 80
      WHEN 4 THEN 95 WHEN 5 THEN 100 ELSE 0 END
    ), 100
  ) AS score
  FROM flood_risk, point
  WHERE ST_Contains(flood_risk.geom, point.geom)
),
school_score AS (
  SELECT LEAST(100, COUNT(*) * 12 +
    (CASE WHEN bool_or(school_type = '小学校') THEN 15 ELSE 0 END) +
    (CASE WHEN bool_or(school_type = '中学校') THEN 15 ELSE 0 END)
  ) AS score
  FROM schools, point
  WHERE ST_DWithin(schools.geom::geography, point.geom::geography, 800)
),
medical_score AS (
  SELECT LEAST(100,
    COUNT(*) FILTER (WHERE facility_type = '病院') * 20 +
    COUNT(*) FILTER (WHERE facility_type != '病院') * 5 +
    LOG(GREATEST(SUM(bed_count), 1)) * 10
  ) AS score
  FROM medical_facilities, point
  WHERE ST_DWithin(medical_facilities.geom::geography, point.geom::geography, 1000)
)
SELECT
  fs.score AS s1_disaster,
  (ss.score * 0.45 + ms.score * 0.55) AS s3_livability,
  fs.score * 0.30 + (ss.score * 0.45 + ms.score * 0.55) * 0.35
    AS tls_mvp
FROM flood_score fs, school_score ss, medical_score ms;
